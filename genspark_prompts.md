# Genspark用プロンプト集

## 戸建（HOUSE_DB）用プロンプト

```
以下の形式で、25件の戸建物件の購入希望者情報を生成してください。各物件には16〜63件のランダムな数の購入希望者情報を含めてください。

【出力形式】
JavaScriptのexport文を含む完全なコードとして出力してください。コピー&ペーストでそのまま使える形式にしてください。

【データ構造】
- 各物件は "name"（物件名）、"address"（空文字列）、"buyers"（購入希望者情報の配列）を持つ
- 各購入希望者情報は以下のフィールドを持つ：
  - "method": 支払い方法（"住宅ローン", "キャッシュ", "キャッシュ（一部住宅ローン）", "未定" のいずれか）
  - "occupation": 職業（空文字列も可。会社員、公務員、教職員、派遣社員、無職、自営業、医師、看護師、エンジニア、デザイナー、営業職、事務職など）
  - "reason": 購入理由（転職のため、結婚を機に、子供の学校区域、老後の住居確保、住み替え、住環境改善、親の近居希望、賃貸から購入へ、投資目的、通勤の便、子育て環境、など）
  - "timing": 購入タイミング（"即時", "1ヶ月後", "3ヶ月後", "6ヶ月後", "1年以内" のいずれか）
  - "ng": NG条件（"特になし", "北向き", "最上階以外", "1階", "ペット飼育歴", "角地以外", "駐車場なし", "日当たり悪い", "交通の便悪い", "築年数古い", など）
  - "family": 家族構成（"ご夫婦", "ご夫婦とお子様", "ご夫婦とお子様二人", "ご夫婦と息子様", "ご夫婦と娘様", "単身", "親と同居", など）
  - "age": 年齢（"30代前半", "40代前半", "50代前半", "購入者：30代前半　居住者：50代後半", "非公開", "20代後半", "60代前半", など様々な形式）

【物件名の例】
- 関西エリアの戸建物件名を想定（例：○○ハイム、○○コーポ、○○ハウス、○○荘、○○邸、など）

【出力例の形式】
```javascript
export const HOUSE_DB_PART = [
  {
    "name": "物件名1",
    "address": "",
    "buyers": [
      {
        "method": "住宅ローン",
        "occupation": "会社員",
        "reason": "結婚を機に",
        "timing": "即時",
        "ng": "北向き",
        "family": "ご夫婦",
        "age": "30代前半"
      },
      // ... 16〜63件の購入希望者情報
    ]
  },
  // ... 25物件分
];
```

【重要】
- 必ずexport文を含めてください
- 各物件のbuyers配列には16〜63件のランダムな数の購入希望者情報を含めてください
- データの多様性を保つため、様々な組み合わせを生成してください
- コピー&ペーストでそのまま使える完全なJavaScriptコードとして出力してください
```

## 土地（LAND_DB）用プロンプト

```
以下の形式で、25件の土地物件の購入希望者情報を生成してください。各物件には16〜63件のランダムな数の購入希望者情報を含めてください。

【出力形式】
JavaScriptのexport文を含む完全なコードとして出力してください。コピー&ペーストでそのまま使える形式にしてください。

- 各物件は "name"（物件名）、"address"（空文字列）、"buyers"（購入希望者情報の配列）を持つ

【物件名の例】
- 関西エリアの土地物件名を想定（例：○○区○○町の土地、○○市○○の分譲地、○○駅前の土地、など）

【出力例の形式】
```javascript
export const LAND_DB_PART = [
  {
    "name": "物件名1",
    "address": "",
    "buyers": [
      {
        "method": "キャッシュ",
        "occupation": "自営業",
        "reason": "新築建て替え",
        "timing": "即時",
        "ng": "角地以外",
        "family": "ご夫婦",
        "age": "50代前半"
      },
      // ... 16〜63件の購入希望者情報
    ]
  },
  // ... 25物件分
];
```

【重要】
- 必ずexport文を含めてください
- 各物件のbuyers配列には16〜63件のランダムな数の購入希望者情報を含めてください
- データの多様性を保つため、様々な組み合わせを生成してください
- コピー&ペーストでそのまま使える完全なJavaScriptコードとして出力してください
- 土地購入の特性を反映した購入理由やNG条件を含めてください
```

## 簡潔版プロンプト（コピー用）

### 戸建用（1行コピー用）
```
25件の戸建物件データを生成。各物件に16〜63件のランダムな購入希望者情報を含める。export const HOUSE_DB_PART = [...]の形式で、コピペでそのまま使えるJavaScriptコードとして出力。各物件は{name:"物件名", address:"", buyers:[{method, occupation, reason, timing, ng, family, age}]}の形式。関西エリアの戸建物件名を使用。
```

### 土地用（1行コピー用）
```
25件の土地物件データを生成。各物件に16〜63件のランダムな購入希望者情報を含める。export const LAND_DB_PART = [...]の形式で、コピペでそのまま使えるJavaScriptコードとして出力。各物件は{name:"物件名", address:"", buyers:[{method, occupation, reason, timing, ng, family, age}]}の形式。関西エリアの土地物件名を使用。土地購入の特性（新築建て替え、投資目的、接道条件、地盤など）を反映。
```

## 使い方

1. 上記のプロンプトをGensparkのAI Chatにコピー&ペースト
2. 生成されたコードをそのままコピー
3. 新しいファイル（例：`data/house/part1.js` または `data/land/part1.js`）に貼り付け
4. 必要に応じて複数回実行してpart2.js, part3.js...を作成

## ファイル保存先

- 戸建データ: `data/house/part1.js`, `part2.js`, ...
- 土地データ: `data/land/part1.js`, `part2.js`, ...
